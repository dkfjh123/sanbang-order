## 현재 로직 (확인된 사실)

| 시점 | 동작 |
|---|---|
| 발주 접수 (pending) | `inventory.quantity -= qty` 즉시 차감 |
| 확정 (confirmed) | 상태만 변경, 재고 손 안 댐 |
| 출고 (shipped) | 상태만 변경, 재고 손 안 댐 |
| 취소 | `inventory.quantity += qty` 복구 |

**관련 코드:**

## 발견된 문제

1. **단일 컬럼이 두 의미 (가용재고 vs 실재고)** 짊어짐. 신화푸드 실재고가 시스템에 없음.
2. **출고 시점에 재고 이벤트 없음.** 물리적 출고가 시스템상으로는 `status` 텍스트 변경뿐.
3. **음수 재고 가드 부재** ([route.ts:279](src/app/api/orders/route.ts#L279)) → 재고 5에 7개 주문 통과됨.
4. **원자성 미흡** — 생성 시 orders/items/inventory/deposit 별도 호출, rollbackAll 부분 처리.
5. **신화푸드 입고 등록 경로 없음** — admin 수동 `inventory_transactions` inbound만 가능.

## A안 — 컬럼 분리 (선택된 방향)

```
inventory:
  on_hand   INT   -- 신화 창고 실재고 (입고/실출고 시에만 변동)
  reserved  INT   -- pending + confirmed 주문 수량 합
  available  GENERATED AS (on_hand - reserved)
```

**타이밍 재배치:**

| 이벤트 | 동작 |
|---|---|
| 발주 (pending) | `reserved += qty` |
| 확정 (confirmed) | 변동 없음 |
| **출고 (shipped)** | **`on_hand -= qty, reserved -= qty`** ← 실재고 이벤트 |
| 취소 | `reserved -= qty` |
| 입고 | `on_hand += qty` |

**시나리오 결과:**
- "재고 10, 5출고대기, 7개 더?" → `available=5` < 7 → 거부, max 5
- "입고100/주문10/출고전" → on_hand=100 (신화 일치), reserved=10, available=90

**기존 데이터 이관 (한 번에 자동):**
```sql
on_hand = quantity + Σ(미출고 주문 수량)
reserved = Σ(미출고 주문 수량)
```
→ `available` 값이 기존 `quantity`와 동일 → 가맹점 화면 안 바뀜.

## 참고: A안 마이그레이션 스켈레톤 (작성 전 초안)

```sql
-- 021_inventory_split_reserved_onhand.sql (미작성)

ALTER TABLE inventory ADD COLUMN on_hand  INT;
ALTER TABLE inventory ADD COLUMN reserved INT DEFAULT 0;

UPDATE inventory i SET
  on_hand = i.quantity + COALESCE((
    SELECT SUM(oi.quantity)
    FROM order_items oi
    JOIN orders o ON o.id = oi.order_id
    WHERE oi.product_id = i.product_id
      AND oi.unit = 'box'
      AND o.status IN ('pending','confirmed')
  ), 0),
  reserved = COALESCE((
    SELECT SUM(oi.quantity) FROM order_items oi
    JOIN orders o ON o.id = oi.order_id
    WHERE oi.product_id = i.product_id
      AND oi.unit = 'box'
      AND o.status IN ('pending','confirmed')
  ), 0);

-- 검증 후 quantity 컬럼 제거 또는 유지 결정
-- pack 단위(낱팩)도 동일 로직 별도 적용 필요 (006 migration 참고)
```

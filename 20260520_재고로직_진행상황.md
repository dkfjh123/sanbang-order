## 오늘 합의된 사항 (확정)

### 1. 재고는 3분할 관리 — A안 채택
| 명칭(사용자 표현) | 시스템 명칭 | 의미 |
|---|---|---|
| 총재고 | `on_hand` | 신화 창고에 실제 박혀 있는 수 |
| 나갈것들 | `reserved` | 미출고 발주 합 (pending + confirmed) |
| 매장주문가능 재고 | `available` | 가맹점이 추가로 주문 가능한 수 (= on_hand - reserved) |

규칙: 가맹점 화면에 보이는 값 = **available**. 기존 단일 컬럼 `inventory.quantity`가 사실상 available 의미였음.

### 2. 차감 시점 변경
- **현재:** 발주 생성(pending) 시점에 즉시 `inventory.quantity -= qty`
- **A안:** 발주 생성 시점에 `reserved += qty` 만. **신화푸드가 출고완료(ship) 누를 때 비로소 `on_hand -= qty, reserved -= qty`** 발생

### 3. 신화푸드(`shinwa`) 권한 — 현재 그대로 유지
- 발주 출고완료 ✅ (이미 가능)
- B2B 출고완료 ✅
- 범용상품 가격/재고 입출고 등록 ✅
- 발주 생성/수정/취소, 전용상품 정보 수정 ❌

→ 권한 변경 불필요. 사용자가 의도한 "신화가 출고완료 눌러야 차감"은 이미 권한상 가능한 상태.

### 4. 음수 가드 — 이미 모든 경로에 박혀 있음
- 발주 신규생성: [orders/route.ts:175-178](src/app/api/orders/route.ts#L175-L178)
- 발주 수정: RPC `update_store_order_items_atomic` (017 마이그)
- B2B 출고: RPC `apply_b2b_inventory_delta` (006 마이그)
- 어드민 재고관리: [inventory/page.tsx:201](src/app/(dashboard)/inventory/page.tsx#L201)

→ 5/19 검토메모의 "음수 가드 부재" 항목은 outdated. 별도 작업 불필요.

### 5. 신화 실측 기반 데이터 보정 (3건)

| 상품 | inventory 변경 | 사유 |
|---|---|---|
| 생밀면 | `quantity 48 → 33` (-15) | 운영 초기 테스트 임시반영 정정 |
| 비빔전용장 | `quantity 2 → 0` (-2), `loose 1 → 0` (-1) | 신화 현장실수 (박스→팩 출고 누락) |
| 육수간장 | `quantity 27 → 26` (-1), `loose 1 → 0` (-1) | 신화 현장실수 (박스→팩 출고 누락) |

각각 `inventory_transactions`에 type='adjustment'로 사유 메모 함께 insert.

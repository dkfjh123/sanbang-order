### 알아둘 갭 (PR2 후속 작업)
- **가맹점 팩 단위 발주 POST/DELETE** — apply_b2b_inventory_delta RPC 가 `reserved_pack` 을 모름. 가맹점 팩 발주 UI 자체가 PR3 에서 도입되므로 그때 같이 손볼 것.
- **B2B 발주 등록(POST)에서 quantity 차감 부재** — 현재 흐름이 SHIP 시점 차감이라 등식은 유지됨(reserved 가 안 늘지만 quantity 도 안 줄어서 OK). 단, 가맹점이 보는 매장주문가능 수치가 약간 부풀려 보일 수 있음(B2B pending 만큼). 향후 B2B POST 도 reserved 패턴 도입 검토.
- **016 같은 수기 반영 SQL** 패턴 — 미래에 inventory 직접 손대는 SQL 쓸 때 `on_hand` 도 같이 손댈 것.
